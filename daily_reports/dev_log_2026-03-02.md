## Technical Learnings

### Q：Rubyのハッシュとparamsハッシュとの違いについて

#### 「ハッシュ」とは

**ハッシュ** は、Rubyのデータ構造の一つで、 **キーと値のペア** を保存するための入れ物

```ruby
# 普通のRubyのHash
user = { "name" => "マッスル", "email" => "example@example.com" }

# シンボルをキーにすることもできる
user = { name: "マッスル", email: "example@example.com" }
```

#### 「paramsハッシュ」とは

Railsでは、ブラウザから送られてきたデータ（フォーム入力、URL内のパラメータなど）を受け取るための **params** という特別なオブジェクト

**なぜ「ハッシュ」と呼ばれるのか？**

**使い方がハッシュとほぼ同じ**

```ruby
# ハッシュのように値を取り出せる
params[:user_id]
params[:status]

# ハッシュのようにネストした構造も扱える
params[:user][:name]
params[:user][:address][:city]
```

#### 具体例

**例1: URLのクエリパラメータ**

```
http://example.com/users?status=activated
```

このURLにアクセスすると、Railsは自動的に以下のような **paramsオブジェクト** を作ります:

```ruby
params = { "status" => "activated" }
```

コントローラでは:

```ruby
def index
  if params[:status] == "activated"
    # 何か処理
  end
end
```

**例2: フォームからのPOSTデータ**

```html
<input type="text" name="user[name]" value="マッスル" />
<input type="text" name="user[email]" value="example@example.com" />
```

このフォームを送信すると:

```ruby
params = {
  "user" => {
    "name" => "マッスル",
    "email" => "example@example.com"
  }
}
```

コントローラでは:

```ruby
def create
  @user = User.new(params[:user])
end
```

---

| **ハッシュ** | Rubyのデータ構造。キーと値のペアを保存する入れ物 |
| --- | --- |
| **params** | Railsが用意する特別なオブジェクト。ブラウザから送られてきたデータを格納する |
| **paramsハッシュ** | paramsオブジェクトの通称。ハッシュのように使えるので「ハッシュ」と呼ばれる |

### Q：form_withは登録するときにどう設計するのか？

- インスタンスオブジェクトをフォームヘルパーにモデルとして渡す。ヘルパーによって自動でHTMLが生成される
- paramsの構造はHTMLで生成される
    - paramsはinputタグにあるnameとvalueによって決まる
    - model:@userのようにモデルのみを渡した時は以下のようにHTML生成。コントローラーでアクセスするときは以下のように記述する
    - 下記は「ネスト構造」とよぶ。

    ```html
    <input name="user[name]" value="太郎">
    params[:user][:name]
    # => "太郎"
    ```

model:@userのみでform_withを使用した場合以下のようにname="user[xxxx]"と生成される。そしてRails側で生成されるparamsハッシュは、「user」をキーとしたparamsハッシュが生成される
userをキーとしてまとめることでコントローラーでアクセスが楽になる

