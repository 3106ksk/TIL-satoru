## 気づき・所感

今日は課題としていたフロントエンド領域の「パーシャルテンプレート」「CSS」系について学ぶことができた日。パーシャルファイルテンプレートをカリキュラムの時はなんとなくで実装していたけど

パーシャルは名前のとおり、「テンプレート」を作る。それをindex.html.erbやnew.html.erbファイルなどのビュー内で使用する。ということが実装意図の言語化をすることができた。

開発中に以下のrenderを使ったオブジェクトの配列内の要素ごとにパーシャルをレンダリングする手法について、「なぜrender」でそれができるのかを深掘りしようとしたが

開発スピードを優先してとりあえず省略記法が行えるRailsの規約条件を理解して終わることにした。

```erb
<% @products.each do |product| %>
  <%= render partial: "product", locals: { product: product } %>
<% end %>

# 省略記法
<%= render partial: "product", collection: @products %>
```

## Technical Learnings

### Q：localsオプションとは?

renderメソッドのオプションとして使用。`locals: {キー名: コレクション}` と使うことでローカル変数としてパーシャルファイルに渡すことができる

### Q：なぜ以下のように省略記法が使えるのか?

```erb
<% @products.each do |product| %>
  <%= render partial: "product", locals: { product: product } %>
<% end %>
```

省略記法:

```erb
<%= render partial: "product", collection: @products %>
```

パーシャルファイル名、ローカル変数、インスタンス変数名が一緒の場合に省略記法が規約によって使うことができる

### Q：daisyUIでリストもしくはカードを作成する方法

公式よりデザインを参照。HTMLのタグにBootstrapの時と同様にCSSクラスを割り当てていく

### Q：Rails環境においてのアセットパイプラインとは

JS、CSS、画像ファイルなどの静的アセットを整理・キャッシュ・配信するために設計されたライブラリのこと

### Q：Tailwind CSSのプラグインとしてdaisyUIの使い方

②Gemfileに以下を追記してインストールを実行

```ruby
gem "jsbundling-rails"
gem "cssbundling-rails"
```

`docker-compose exec web bundle install` をターミナルで入力し、bundle install

③ `docker-compose exec web bundle exec rails css:install:tailwind` を実行

④yarnでdaisyUIをインストール

`docker-compose exec web yarn add daisyui` をターミナルで入力して実行

⑤tailwind.config.jsの作成と編集
