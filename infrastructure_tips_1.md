# 未経験が考えるべきインフラ構成

ちゃんとRUNTEQで見てきた中で評価されるというか、ここまでやっておけば問題ないという現実的なレベルのポートフォリオインフラ構成を示します。

- **Web**: Render or Fly.io
- **DB**: Neon
- **キャッシュ**: Upstash
- **画像**: S3 or Cloudinary
- **メール配信（MTA）**: Mailgun
- **DNS、メール受信**: Cloudflare
- **CI/CD**: GitHub ActionsでLintとCIを設定し、mainマージにて自動でPaaSにデプロイ（PaaS側で設定）

これが個人サービスを最小コストで運用するための一例の構成です。この構成のメリットはそれぞれ負荷が高まった時に個別課金してスケールアップもしくは移管しやすいことです。スケールアウトはその後でよいです。

## 実務に合わせたAWS構成の場合

ちなみに実務に合わせたAWSの構成を作るのであれば：

`GitHub Actions -> ECR -> EventBridge -> (CodeBuild) -> ECS (Fargate上)`

こんな感じでコンテナベースのDevOpsを組んでいく必要があり、未経験には難しいので避けるべきかなとも思ってます。それよりも今は素早く低コストで価値を届ける方が重要です。
