# Daily Review — Notion入力テンプレート

> Notionの Daily Review ページ本文にこの構造で記入する。
> `format_daily_log` スキルがこのフォーマットを前提に整形する。

## テンプレート（ここからコピー）

```
Done条件:
切れたら→:

---

📝 Context & Reflection

（今日やったこと・プロセスを自由に書く）

### ✅ Worked
-

### 🚧 Slipped
-

### 💡 Insights
-

### 📅 Study Strategy for Next Day
-
```

## 記入ガイド（各セクションの勘所）

**Done条件 / 切れたら→**
- 未記入だとDaily Planから補完するが、実際の実行時と乖離する場合がある
- 一言でOK。例: `Done条件: 20:05までに3ステップ書き出し` / `切れたら→: Railsガイド読む`

**Context & Reflection**
- 自由記述でOK。「何やった」「どこまで進んだ」「詰まった箇所」が含まれていると整形精度が上がる
- 技術的なQ&Aを書く場合は **Q: / A:** を対にすると Technical Learnings に自動転記される（Aがない場合は省略）

**Worked / Slipped / Insights**
- 一行でもいいので**具体的な行動・事実**を書く。例: `昼休憩に24min確保できた` / `19時以降集中切れた`
- プレースホルダー（`[工夫など]`）のまま残すと、AIがセッションnotesから全推測する（精度低下の原因）
- Insightsは仮説レベルでOK。「〜かもしれない」で十分
