## 2. 1日の行動基本設計

### 2-1. Off Day テンプレート（学習可能時間: 5-6h）

```
07:30  起床・朝食・身支度
08:00  ┌─ Morning Planning (15min)
       │  ・逆算型ゴール設定（週次マイルストーンから当日Top1を決定）
       │  ・Top1 + 「切れたら→」タスクを設定
       │  ・「これは週次マイルストーンに繋がっているか？」を確認
       └─
08:15  ┌─ AM Block: 高負荷タスク (3.0-3.75h)
       │  Cognitive Headroom: 4-5
       │  ──────────────────────
       │  [Phase 1] 設計・実装（DB設計、CRUD、機能実装）
       │  [Phase 2] 実装継続 or 技術書Reading（体系的インプット）
       │  ──────────────────────
       │  ルール:
       │  ・12:00アラームで強制切り替え（完璧主義防止）
       │  ・ハマり30分でアプローチ変更 or 技術面談予約
       └─
12:00  昼食・休憩 (60min)

13:00  ┌─ PM Block: 中負荷タスク (2.0h)
       │  Cognitive Headroom: 3-4
       │  ──────────────────────
       │  [Phase 1] 実装継続、バグ修正
       │  [Phase 2] Reading（技術記事、公式ドキュメント）or 記事執筆
       │  ──────────────────────
       │  ルール:
       │  ・「切れたら→」トリガー: 自己感知 or 15:00アラーム
       │  ・15:00で集中が切れた場合 → Readingに切替
       └─
15:00  散歩 (45min) ※認知リセット

15:45  ┌─ Evening Block: 中〜低負荷タスク (1.5h)
       │  Cognitive Headroom: 3
       │  ──────────────────────
       │  [Phase 1] 軽い実装、ドキュメント確認
       │  [Phase 2] 記事執筆（日曜）or Reading or 技術面談Prep
       │  ──────────────────────
       └─
17:30  夕食・自由時間

21:00  ┌─ Night Review (15min)
       │  ・事実記録のみ（Daily Reviewの「Fact」セクション）
       │  ・翌朝に「なぜ」「Insight」を加筆（15-20min）
       │  ・明日のTop1を仮設定
       └─
21:15  自由時間・就寝準備

───────────────────────────
合計学習時間: 約5.5-6.0h (330-360min)
DS記録: 各セッションで記録
───────────────────────────
```

### 2-2. Shift Day テンプレート（学習可能時間: 1-2h）

```
出勤前 ┌─ 朝5min計画 (5min)
       │  ・帰宅後の1タスクを決定（If-Then）
       │  ・「帰宅したら、まず[タスク名]をする」を明示
       └─

       出勤 → 勤務 → 帰宅

帰宅後 ┌─ Immediate Block (30-40min)
       │  If-Then: 「帰宅 → 荷物を置く → 即Reading」
       │  ──────────────────────
       │  [Phase 1] 技術ドキュメント確認、軽い設計メモ
       │  [Phase 2] 技術書Reading（30min固定）
       │  ──────────────────────
       │  Done条件: タイミングベース（「帰宅後すぐ30分」）
       └─

       ┌─ Light Block (30-40min) ※余力がある場合
       │  ・軽い実装、コードレビュー
       │  ・記事のネタメモ作成
       │  ・技術面談Prep（翌日面談がある場合）
       └─

21:00  ┌─ Night Review (10min)
       │  ・事実記録のみ（Shift Day版: 最低3行）
       │  ・翌朝に加筆
       └─

───────────────────────────
合計学習時間: 約1.0-1.5h (60-90min)
最低保証: Reading 30min
───────────────────────────
```

### 2-3. 日曜テンプレート（記事執筆日）

```
Off Day テンプレートに以下を上書き:

13:00  ┌─ PM Block → 記事執筆 (2.0-3.0h)
       │  Cognitive Headroom: 3-4
       │  ──────────────────────
       │  ・その週の実装 or 読書で学んだことを記事化
       │  ・構成: 課題 → 仮説 → 行動 → 学び
       │  ・2000-3000字を目安
       │  ──────────────────────
       │  ルール:
       │  ・金曜までにミニアプリの実装を区切る
       │  ・記事テーマは週次で事前決定済み
       └─

───────────────────────────
日曜のポイント:
・AM Block = 実装 or Reading（通常通り）
・PM Block = 記事執筆（固定枠）
・Evening Block = 自由（書けなかった分の記事 or Reading）
───────────────────────────
```
