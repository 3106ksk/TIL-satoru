## ■ Technical Learnings

**1. Rails: ルーティングのリソース命名規則**
- **Question**: ユーザー登録時のルーティングのリソースは複数形？単数形？
- **Answer**: 複数形（`resources :users`）。ユーザー登録は複数のユーザーが存在し得るリソースであるため

**2. Rails: アカウント作成に必要なアクション**
- **Question**: アカウント作成にはどんなアクションが必要になるのか？
- **Answer**: 3つのアクションが必要
  - `new`: フォームヘルパーでテーブル属性に対応するparamsを受け取るための初期化
  - `create`: ブラウザから送信されたparamsをDBに保存
  - Strong Parameters: 必要な属性がブラウザから送信されているかをチェックするフィルタリング機構

**3. Ruby/Rails: ハッシュとparamsハッシュの違い**
- **Question**: Rubyのハッシュとparamsハッシュとの違いは？
- **Answer**: ハッシュはRubyのデータ構造（キーと値のペア）。paramsはRailsが用意する`ActionController::Parameters`型の特別なオブジェクトで、ブラウザからの外部入力を格納する。操作方法はハッシュと同一だが、セキュリティ機構（Strong Parameters）が組み込まれている
- **Code**:
  ```ruby
  # 普通のRubyのHash
  user = { name: "マッスル", email: "example@example.com" }

  # paramsハッシュ（ハッシュのようにアクセス可能）
  params[:user][:name]       # ネスト構造でアクセス
  params[:user][:email]
  ```

**4. Rails: form_withのモデル渡しとparams生成**
- **Question**: form_withは登録するときにどう設計するのか？
- **Answer**: インスタンスオブジェクト（`@user`）をモデルとして渡すと、属性に対応するHTML inputタグが自動生成される。paramsの構造はHTMLのname属性（`user[name]`形式）によって決定され、コントローラーでは`params[:user][:name]`でアクセスする
- **Code**:
  ```erb
  <%# form_withにモデルを渡す %>
  <%= form_with model: @user do |f| %>
    <%= f.text_field :name %>
    <%# => <input name="user[name]"> が生成される %>
  <% end %>
  ```
  ```ruby
  # 生成されるparamsハッシュ
  # { "user" => { "name" => "らんてくん", "email" => "example@example.com" } }

  # コントローラーでのアクセス
  params[:user][:name]  # => "らんてくん"
  ```
- **Evidence**: https://railsguides.jp/action_controller_overview.html#strong-parameters / https://qiita.com/hinanata72/items/11b4b4be3a5572f3a102
