**■Key Insight（今日のハイライト学び1つ）**: 後で書く（任意、1行）

Promise/非同期処理の大枠の理解について

- 外部のアプリやサーバーなどにアクセスをして結果を待ちながら他の処理ができる→UI/UXの向上にとても重要
- 処置中、成功(resolve)、失敗(reject)の3つの状態によって処理をコントロールする。➡️ オブジェクトにstate: として格納されいた
- new Promise()をなぜ行うのか？ ：Promiseオブジェクトを作成
- executor関数とは？：new Promiseに格納される関数

JSの標準機能によって resolve,rejectの関数が作成。下記のresolve,rejectは関数を受け取るためのパラーメータ←極論なんでも良い

```jsx
(resolve, reject) => {
  setTimeout(() => {
    resolve("成功しました！");
  }, 2000);
}
```

new Promiseで作成されたオブジェクトと.then()メソッドの関係について理解のために

console.log("成功時:", myPromise);で検証をしてみた

- resolve()：を使うことで成功時の処理が実行されて、下記のobjectが生成

```jsx
const myPromise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("成功しました！");
  }, 2000);
});

// Promiseの結果を処理
myPromise
  .then((result) => {
    console.log("成功時:", myPromise);
    console.log("Promiseの結果:", result);
  })
  .catch((error) => {
    console.log("こっちは実行されないはず:", myPromise);
    console.error("Promiseのエラー:", error);
  })
  .finally(() => {
    console.log("Promiseの処理が完了しました");
  });

console.log("処理終了");

```
