**1. Ruby: ハッシュ引数の省略記法**
- **Question**: `User.exists?(reset_password_token:)` の引数のシンボルについてなぜこの書き方なのか？
- **Answer**: メソッドの引数にハッシュを使うときの省略記法で、keyとvalueの変数名が同じ時に省略が可能（Ruby 3.1+）。
- **Code**:
  ```ruby
  # 以下は全て同じ意味
  User.exists?(reset_password_token: reset_password_token)  # フル記述
  User.exists?({ reset_password_token: reset_password_token })  # ハッシュとして明示
  User.exists?(reset_password_token:)                       # 省略記法
  ```

**2. Rails: ActiveRecordのカラムアクセス**
- **Question**: モデルに対してのメソッド呼び出しとカラムアクセスの区別はどうやって行うか？（`self.reset_password_token = reset_password_token`がメソッドかカラム代入か）
- **Conclusion**: Railsの `ActiveRecord` は、DBのカラムに対して **自動的にgetter/setterメソッドを生成**する。そのため、カラム名でメソッドとしてアクセス可能。
- **Evidence**: https://zenn.dev/wwwave/articles/2e80f16aac8974
- **Code**:
  ```ruby
  content = Content.find(21)
  content.price # 自動生成されたメソッドを通じてカラム値にアクセス
  ```
