### Rails: Ransackの実装要件読み解き
- **Question**: 掲示板検索機能を実装する際、カリキュラム要件を満たすにはどのセクションを読むべきか？
- **Conclusion**: 基本的な実装はSimple Mode、ViewはFormHelperを参照する。ドキュメントのPersonクラスをBoardに置き換えて実装する。
- **Code**:
  ```ruby
  # Controller example
  def index
    @q = Person.ransack(params[:q])
    @people = @q.result(distinct: true)
  end
  ```

### Rails: Ransack検索フロー
- **Question**: 検索のフローを言語化すると？
- **Conclusion**: 1) `search_form_for`で条件設定、2) `params[:q]`でクエリ生成、3) 結果表示。
- **Code**:
  ```ruby
  # Debugging @q object
  [1] pry(#<BoardsController>)> @q
  => Ransack::Search<class: Board, base: Grouping <conditions: ... >>
  ```

### Rails: Ransackの空検索挙動
- **Question**: なぜ検索キーワードを設定しなくてもエラーにならないのか？
- **Cause**: `params[:q]`が`nil`でもRansackは空の検索オブジェクト（全件取得）を生成するため。
- **Conclusion**: 検索キーワードがない場合は全件取得となる仕様である。
- **Evidence**:
  ```ruby
  @q = Board.ransack(params[:q])
  # params[:q] is nil -> Returns all records
  ```
