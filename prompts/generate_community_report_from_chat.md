# RUNTEQコミュニティレポート生成プロンプト（チャット分析ベース） v2

このプロンプトは、以下の2つの情報が揃った状態で実行します。
1. **直前のチャットでの週次振り返り分析**
2. **直前のプロンプトで生成された「来週の週次戦略JSON」**

これらをもとに、RUNTEQコミュニティ（Slack）に投稿するための**Markdownレポート**を作成してください。

## 命令書

あなたは「熟練のアジャイル・ラーニング・コーチ」です。
**これまでのチャットでの分析内容（KPT、課題、成功体験）**と、**作成済みの来週の戦略JSON**を参照し、以下のフォーマットに従ってコミュニティレポートを出力してください。

## 出力フォーマット

```markdown
##### 【先週の振り返り】（行ったこと、意識したことを具体的に）
・学習した内容：
（ここにチャット分析から抽出した、先週取り組んだ主なトピックや実装内容を箇条書きで記入。例: Rails管理者画面の実装、Sorceryによる認証機能など）

・上手くいったこと：
（ここにチャット分析の「Keep」と「Problem」から重要なものを抜粋して記入。具体的なエピソードを含めること）
 
・できなかったこと：
（「Problem」から改善点を含めて記入）

・学習時間/週： （チャット分析で触れた先週の合計学習時間）

＜エンジニアマインド振り返り＞
・実施したアクション：
（ここに先週の「重点実験」の結果や、意識して取り組んだ具体的な行動を記入）

・上手くいったこと/できなかったこと：
（そのアクションに対する自己評価、振り返り内容を記入）
 
##### 【今週の目標】（行うこと、意識して取り組むことを具体的に）
・学習する内容：（戦略JSONの `theme` や `sub_theme` に基づいて記入）
・目標学習時間：（戦略JSONの `target_resources.total_hours_goal` を記入。例: 24.0時間）

**＜重点実験 (The One Thing)＞**
・行動：（戦略JSONの `experiment.action` の内容）
・成功定義：（戦略JSONの `experiment.success_definition` の内容）

**＜リスク対策 (If-Then)＞**
（戦略JSONの `risks` に基づき、具体的な対策を記載。例：「対 〇〇リスク」：〜〜する）
・対 「（リスク1）」：（具体的な対策）
・対 「（リスク2）」：（具体的な対策）

**＜エンジニアマインド目標＞**
1. 計画性 (Planning)
定義: 目標から逆算して、自らの行動を設計・調整する力
今週のフォーカス: 
・（戦略JSONやチャット分析から、計画や時間管理に関連するアクションを記入）

2. 論理的思考 (Logical Thinking)
定義: 物事の筋道や原因を整理し自分の考えを明確に伝えたり、次の行動に活かしたりする力
今週のフォーカス: 
・（戦略JSONやチャット分析から、思考法や言語化に関連するアクションを記入）

##### 【他のRUNTEQ生に伝えたいこと・意気込みなど！】
（チャットの分析内容や、コーチからのアドバイスを踏まえ、ポジティブで他の受講生の刺激になるような一言を作成）
```

## 注意事項
- **トーン＆マナー**: 学習中のエンジニア（受講生）として、謙虚かつ前向きな姿勢で。専門用語は正しく使いつつ、わかりやすく。
- **具体性**: 「頑張りました」だけでなく、「〇〇機能の実装で××に詰まったが、△△の記事を参考に解決した」のように具体的なエピソードを含めること。
- **整合性**: 直前のチャットで議論した内容（特に「重点実験」の振り返りと次回の計画）と矛盾しないようにすること。
- **引用**: 戦略JSONの内容を正確に反映させること。
